S.I.R.E. – Simulated Incident Response Environment

Problem: Emergency response teams need realistic, repeatable training scenarios that can be run remotely and scaled to different complexity levels — but existing tools are either cost-prohibitive or too static to support live instructor-driven escalation.

Solution: Designed and built the complete backend for a real-time, instructor-led incident response simulator. The platform delivers low-latency event streaming via Socket.IO, a modular scenario engine driving 8 escalating simulations, a secure REST API, and multi-layer security (rate limiting, helmet, input validation). Deployment tooling targets Docker, Railway, Render, and Heroku.

Technologies: Node.js (ESM), Express, Socket.IO, Docker, OpenAPI, CI/CD (GitHub Actions), cloud deployment (Railway / Render / Heroku).

Esoteric Wisdom – Spiritually Themed Web Platform

Problem: Building a fully-featured, secure web application in a systems programming language is a steep learning curve — most web frameworks favor dynamically typed languages, leaving performance-focused alternatives underexplored by junior developers.

SARCAN United — Canadian SAR Digital Operations Platform

Problem: Canada’s Search and Rescue operations span land, air, and maritime domains across federal, provincial, and volunteer jurisdictions — yet teams typically rely on disconnected, paper-based, or proprietary tools that create critical interoperability gaps during time-sensitive incidents. No unified, open-source platform existed that aligned with the CAMSAR Manual, IAMSAR, and the National Search and Rescue Program (NSP).

Solution: Designed and built an open-source, nationally interoperable SAR operations platform covering all three domains. Key capabilities include multi-domain GIS mapping (topographic, satellite, OpenSeaMap, live precipitation radar, weather overlays), IAMSAR-aligned probability modeling (POA/POD/POS heatmaps, coverage factor, effective sweep width, 95% containment radius), 14 NASAR lost-person behaviour profiles with automatic IPP radius drawing, an air search-pattern generator (parallel-track and expanding-square), maritime drift datum visualization, a full Canadian ICS and SAR forms library (ICS 201–214, MPQ, SAA, IAP), a public emergency SOS and trip-plan interface, and simulation/training scenarios with instructor injects. Supports federal (CAF, CCG, JRCC), provincial (SARVAC, OSARVA), and volunteer organizations. A candidate technology for the SAR New Initiatives Fund (SAR NIF).

Technologies: HTML, CSS, JavaScript, Leaflet.js, OpenTopoMap, OpenSeaMap, OpenWeatherMap, RainViewer, Esri basemaps, GPX export, role-based authentication (IC / Planning / Operations / Field / Air / Public Liaison), immutable audit log.
